Análise Forense: Campanha FAUX#ELEVATE e o Sequestro de Recursos Corporativos

Publicado por Equipe Técnica Visual Studio & Security | 24 de Março, 2026

Hackers estão utilizando uma nova abordagem altamente sofisticada para comprometer redes empresariais, focando no roubo massivo de credenciais e na implementação de mineradores de criptoativos ocultos[cite: 1]. Esta operação, identificada por especialistas como FAUX#ELEVATE, utiliza engenharia social refinada e abusos de infraestruturas legítimas para contornar defesas tradicionais[cite: 2].

A Anatomia do Ataque: O VBScript de 224 Mil Linhas

O ataque inicia-se com e-mails de phishing direcionados que entregam currículos (CVs) falsos[cite: 1, 3]. Quando o arquivo é aberto, o sistema exibe uma mensagem de erro simulando um arquivo corrompido — um "bait" clássico[cite: 3]. Por trás dessa tela, o Windows executa um script Visual Basic Script (VBS) massivo com aproximadamente 224 mil linhas de código ofuscado, das quais apenas 226 contêm a carga útil real[cite: 3].

Exfiltração via SMTP e Roubo de Credenciais

O malware utiliza o projeto ChromElevator para contornar as proteções de criptografia vinculada a aplicativos (ABE) e extrair senhas de navegadores baseados em Chromium[cite: 5]. Além disso, ferramentas específicas como o mozilla.vbs e o walls.vbs são implantadas para roubar perfis do Firefox e arquivos sensíveis diretamente do desktop[cite: 6].

Os dados roubados são exfiltrados através de duas contas russas independentes (olga.aitsaid@mail.ru e 3pw5nd9neeyn@mail.ru) que compartilham a mesma senha via protocolo SMTP, enviando o conteúdo para o endereço final do atacante[cite: 6, 7].

Criptomineração e Persistência no Kernel

Após o roubo de dados, a campanha implementa o mservice.exe (XMRig) para minerar Monero[cite: 1, 6]. Para garantir o desempenho máximo, é utilizado o driver de kernel WinRing0x64.sys, permitindo o sequestro total do potencial da CPU[cite: 6]. O Trojan RuntimeHost.exe mantém a persistência, modificando regras de firewall e comunicando-se periodicamente com servidores de comando e controle (C2) hospedados em sites WordPress comprometidos[cite: 2, 6].

Alerta de Velocidade: A cadeia de infecção completa — da execução do VBS até a exfiltração final — ocorre em aproximadamente 25 segundos, tornando a resposta manual humana praticamente impossível[cite: 10].

Mecanismo de Defesa Recomendado

Para mitigar esta ameaça em sua rede, nossa equipe recomenda a aplicação imediata do seguinte script de endurecimento via PowerShell, focado em desativar o Windows Script Host e bloquear as portas de exfiltração SMTP:

# Desativa o motor de execução VBScript malicioso
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows Script Host\Settings" -Name "Enabled" -Value 0

# Bloqueia exfiltração via SMTP para e-mails desconhecidos
New-NetFirewallRule -DisplayName "Block_Malware_SMTP" -Direction Outbound -LocalPort 587 -Protocol TCP -Action Block

Referência: Dados baseados no relatório técnico da Securonix sobre operações de ameaças em múltiplas etapas[cite: 9, 10].
